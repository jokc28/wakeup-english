# 옥모닝 — 민감 권한 사유서 (Play Console / 사용자용)

검토일: 2026-05-15. Play Console의 사전 신고 권한 사유 입력란과 앱 내 권한 요청 다이얼로그
문구에 모두 동일하게 사용한다.

## 1. SCHEDULE_EXACT_ALARM / USE_EXACT_ALARM

- **사용 사유**:
  옥모닝은 사용자가 지정한 시간에 정확히 알람이 울려야 하는 알람 시계 앱이다.
  Android 12 이상에서 비정확 알람을 사용할 경우 OS가 알람 시간을 최대 ±15분
  지연시킬 수 있어 알람 시계의 핵심 기능이 손상된다. 따라서 정확한 시각에 알람이
  발동되도록 `SCHEDULE_EXACT_ALARM`/`USE_EXACT_ALARM` 권한이 반드시 필요하다.

- **사용 빈도**: 사용자가 알람을 생성/수정할 때마다.
- **대체 수단 검토**: WorkManager 등 비정확 스케줄러는 알람 시계 용도로 부적합 — 채택 불가.

## 2. USE_FULL_SCREEN_INTENT

- **사용 사유**:
  알람이 발동되면 잠금 화면 위에서 영어 퀴즈를 풀어야 알람이 꺼지는 구조이다.
  잠금 상태 또는 다른 앱 사용 중에도 화면 전체로 알람 UI를 즉시 띄우기 위해
  필요하다.

- **사용 빈도**: 알람 트리거 시점에만.

## 3. RECORD_AUDIO (마이크)

- **사용 사유**:
  말하기 미션에서 사용자의 발음을 받아 단어/표현 정답 여부를 채점하기 위함이다.
- **데이터 흐름**:
  Android `SpeechRecognizer`를 통해 발화를 텍스트로 변환한 결과만 앱에서 사용.
  옥모닝 자체 서버로는 음성 또는 텍스트를 송출하지 않음.
- **사용 빈도**: 사용자가 말하기 미션을 선택했을 때만 활성화. 권한 거부 시
  말하기 미션을 비활성화하고 다른 미션 유형(단어 정렬, 받아쓰기)으로 폴백.

## 4. POST_NOTIFICATIONS

- **사용 사유**: 알람 트리거 알림, 학습 리마인더 표시.
- **거부 시 동작**: 알람은 풀스크린 인텐트로 여전히 발동되지만 보조 알림은 표시되지 않는다.

## 5. FOREGROUND_SERVICE / FOREGROUND_SERVICE_MEDIA_PLAYBACK

- **사용 사유**:
  사용자가 알람 음원을 멈추지 않은 채로 다른 앱으로 전환하더라도 알람이 계속
  울리도록 OS가 프로세스를 종료하지 않게 보장. `mediaPlayback` 타입은 알람음
  재생이 미디어 재생 카테고리에 해당하기 때문에 사용한다.

## 6. WAKE_LOCK / DISABLE_KEYGUARD / RECEIVE_BOOT_COMPLETED

- **사용 사유**:
  - `WAKE_LOCK`: 알람 발동 중 단말이 슬립으로 들어가지 않도록.
  - `DISABLE_KEYGUARD`: 잠금화면 위에 알람 UI를 표시.
  - `RECEIVE_BOOT_COMPLETED`: 재부팅 후 예약된 알람을 다시 등록.

## 7. REQUEST_IGNORE_BATTERY_OPTIMIZATIONS

- **사용 사유**:
  Doze 모드에서 알람이 누락되지 않도록 사용자에게 배터리 최적화 예외를 요청.
- **주의**: 사용자가 거부해도 앱은 동작하지만, 새벽 시간 알람 누락 가능성이
  생긴다. 권한 요청 다이얼로그에서 이 트레이드오프를 명시.
